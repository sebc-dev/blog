# Epic 1: Initialisation et Infrastructure du Blog (Core Platform Setup)

**Version:** 0.1
**Date :** 2025-05-11
**Statut :** Ébauche

## 1. Objectif de l'Epic

L'objectif principal de cet Epic est de mettre en place l'ensemble de l'infrastructure technique de base nécessaire pour héberger et servir le blog technique bilingue. Cela inclut la configuration du serveur VPS, l'installation des technologies de conteneurisation, la mise en place du reverse proxy, la structure initiale des projets frontend et backend, et les configurations de base pour la base de données et le thème visuel. À la fin de cet Epic, nous devrions avoir un "shell" fonctionnel de l'application, prêt à recevoir le contenu et les fonctionnalités plus spécifiques.

_(Réf PRD : "Epic 1: Initialisation et Infrastructure du Blog (Core Platform Setup) - Goal: Mettre en place la plateforme de blog avec la stack technique spécifiée (Astro pour le frontend/MDX, Spring Boot pour les API de comptage backend, PostgreSQL), configurer l'hébergement VPS (OVH, Debian, Docker, Traefik), le thème de base responsive avec TailwindCSS et DaisyUI, et les fonctionnalités essentielles non liées au contenu (sécurité de base avec Traefik, performance initiale).")_

## 3. Critères d'Acceptation Généraux pour l'Epic

-   Le VPS est configuré, sécurisé (SSH par clé durci, `iptables-nft`, `fail2ban`, `unattended-upgrades`) et Docker/Docker Compose sont installés.
-   Traefik est déployé sur le VPS et capable de router le trafic HTTP/HTTPS, avec génération de certificats Let's Encrypt fonctionnelle.
-   La base de données PostgreSQL est déployée dans un conteneur Docker avec persistance des données.
-   Le projet monorepo est initialisé avec la structure de base (`frontend/`, `backend/`, `docs/`).
-   L'application frontend Astro peut être buildée en une image Docker (avec Nginx) et servie. Une page d'accueil squelette avec le thème de base (clair/sombre) est accessible.
-   L'application backend Spring Boot peut être buildée en une image Docker et démarrée. Elle se connecte à la base de données PostgreSQL et expose un endpoint de santé Actuator (`/actuator/health`) basique.
-   Un fichier `docker-compose.yml` permet de lancer l'ensemble des services (frontend, backend, db, traefik) en environnement de développement local.
-   Un fichier `docker-compose.prod.yml` est prêt pour le déploiement en production sur le VPS (utilisant des images de GHCR).
-   La documentation de base pour la structure du projet, la configuration de l'environnement de dev, et les variables d'environnement est initiée.

## 4. Dépendances

-   Accès au VPS OVH existant.
-   Accès à un dépôt Git (GitHub).
-   Choix des versions technologiques validés (voir `teck-stack.txt`).

## 5. Risques et Questions Ouvertes

-   **Sécurisation de Traefik et de ses configurations (ACME) :** Assurer les bonnes permissions et la sauvegarde.
-   **Configuration du réseau Docker pour Traefik :** S'assurer que Traefik peut découvrir et router correctement vers les services applicatifs sans les exposer directement.
-   **Choix définitif du serveur web pour l'image Docker Astro (Nginx vs Caddy) :** Nginx est retenu pour l'instant. (Point de la `TODO.txt`)
-   **Gestion des variables d'environnement entre local, CI, et production :** Le fichier `.env` sur le VPS est la source de vérité pour la prod.

## 6. Hors Scope pour cet Epic

-   Implémentation complète des fonctionnalités métier des Epics suivants (gestion de contenu, API de métriques complète, SEO avancé, CI/CD complet).
-   Création du contenu initial du blog.
-   Configuration avancée de l'observabilité (monitoring détaillé, alerting).
